## 버전 관리 정책

### 버전 번호 체계
- **Major.Minor.Patch** 형식 사용
- **Major**: 대규모 변경, 하위 호환성 없음
- **Minor**: 새로운 기능 추가, 하위 호환성 유지
- **Patch**: 버그 수정 및 작은 개선

### 릴리스 주기
- 주요 업데이트: 필요 시
- 버그 수정: 즉시 대응
- 성능 개선: 정기적으로
